  - 따로 발표를 위해 슬라이드를 만들 필요는 없으며, 논문을 따라 보거나, 블로그 글, 저자의 슬라이드 등 가용 가능한 자료들을 자유롭게 활용함.
- 스터디 내용은 **논문**에만 국한되어 있지 않고, **프레임워크/소스코드**나, **블로그** 글, 또는 **본인이 진행하고 있는 연구/논문**을 소개할 수 있음.
  - 발표자의 연구/논문에 조언을 한다고 해서 저자에 포함되는 것은 아니기 때문에 부담없이 서로의 의견을 묻고 답함.
- 스터디 발표는 본 레포지토리에 이슈를 새로 생성하여 사전 자료나 discussion 내용, 코멘트 등을 기록하고 (전부 optional) 스터디가 끝나면 close함.
- 공지나 기타 간단한 커뮤니케이션은 슬랙 채널을 통해 이루어짐.

### 학회 스프린트
- 매년 major ML 학회의 논문 리스트가 공개 되었을 때, 한 시간 정도 자리에 모여 읽을 만한 논문들을 빠르게 추리는 작업.
- 최대한 각자 관심이 가는 논문들을 빠르게 skim reading하여 점수를 메기고, 간단하게 코멘트를 작성함.
- 모두의 점수들로 논문들을 정렬하고, 논문들을 간단하게 소개함.
  - 어떤 문제를 어떻게, 어떤 성능으로 풀었고, 왜 이 점수를 주었는지 등.
